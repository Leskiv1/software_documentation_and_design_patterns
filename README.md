
# Steam Platform Data Management System (Лабораторна 3)

Проєкт для автоматизації генерації, обробки та імпорту денормалізованих даних ігрової платформи (користувачі, ігри, ігрові сеанси, досягнення) з CSV-файлу в реляційну базу даних. Додатково реалізовано повноцінний веб-інтерфейс на основі патерну **MVC (Model-View-Controller)** для виконання CRUD-операцій над основною сутністю ("Гра").

Проєкт побудований з використанням суворої трирівневої архітектури, шаблонів **Інверсія управління (IoC)** та **Впровадження залежностей (DI)**.

## Технології

  * **Мова:** Python 3.10+
  * **Framework:** FastAPI (веб-сервер, DI-контейнер та маршрутизація)
  * **Шаблонізатор (View):** Jinja2 (генерація HTML-сторінок)
  * **UI/UX:** Bootstrap 5 (стилізація форм та таблиць)
  * **ORM:** SQLAlchemy 2.0+
  * **База даних:** SQLite (вбудована, драйвер `sqlite3`)
  * **Формат даних:** єдиний денормалізований `.csv` файл (1000+ рядків)
  * **Генерація даних:** Faker
  * **Сервер:** Uvicorn

## Архітектура проєкту (Layered Architecture + MVC)

Проєкт розділений на незалежні шари. Бізнес-логіка повністю ізольована від деталей реалізації бази даних та веб-фреймворку за допомогою інтерфейсів. Для веб-частини імплементовано класичний патерн MVC.

### Структура шарів:

**1. DAL (Data Access Layer / Model) — Рівень доступу до даних (`data_access/`)**

  * `interfaces.py`: Абстрактні класи (`IDataLoader`, `IPlatformRepository`), які визначають контракти. Розширено методами для CRUD-операцій.
  * `repositories.py`: Інкапсулює логіку запитів до бази даних через SQLAlchemy ORM (парсинг файлів, додавання, редагування, видалення та пошук ігор).
  * `csv_reader.py`: Відповідає за читання сирих даних з файлової системи.
  * `models.py`: ORM моделі таблиць бази даних із налаштованими зв'язками (Foreign Keys).

**2. BLL (Business Logic Layer) — Рівень бізнес-логіки (`business/`)**

  * `services.py`: Головний сервіс `SteamPlatformService`. Координує процеси: розв'язує денормалізовані рядки з CSV, керує створенням об'єктів, а також обробляє запити від веб-контролерів для маніпуляції сутністю "Гра". **Залежить виключно від абстрактних інтерфейсів.**
  * `dependencies.py`: Налаштування Dependency Injection за допомогою механізму `Depends()` з FastAPI.

**3. Presentation Layer (Controllers & Views) — Презентаційний рівень**

  * `presentation/routers.py`: API ендпоінти для автоматизованого завантаження даних (Лаб 2).
  * `presentation/web_routers.py` **(Controller)**: Контролери веб-додатку. Приймають HTTP-запити з браузера, звертаються до Моделі через бізнес-сервіс та повертають HTML-сторінки.
  * `templates/` **(View)**: HTML-шаблони, які відповідають за візуалізацію даних (каталог ігор, форми додавання та редагування).
  * `main.py`: Точка входу, ініціалізація бази даних та підключення обох типів роутерів.

## Інструкція із запуску

### 1\. Встановлення залежностей

Переконайтеся, що ви знаходитесь у віртуальному середовищі проєкту. Встановіть всі необхідні бібліотеки з файлу `requirements.txt`:

```bash
pip install -r requirements.txt
```

### 2\. Генерація тестових даних

Для первинного наповнення програми згенеруйте CSV-файл (понад 1000 рядків), який з'явиться у папці `data/`:

```bash
python data_generator/generate_csv.py
```

### 3\. Запуск сервера

Запустіть основний додаток за допомогою Uvicorn. База даних `steam_app.db` створиться автоматично під час першого запуску:

```bash
python main.py
```

### 4\. Автоматизований імпорт даних (Лаб 2)

Для масового завантаження даних з CSV-файлу у БД відкрийте **новий термінал** і виконайте клієнтський скрипт:

```bash
python client_test.py
```

*Скрипт відправить запит на API, який ініціює парсинг та розподіл даних по таблицях.*

### 5\. Робота з веб-інтерфейсом MVC (Лаб 3)

Відкрийте браузер та перейдіть за адресою:
**[http://127.0.0.1:8000/games](https://www.google.com/search?q=http://127.0.0.1:8000/games)**

Через зручний UI ви можете:

  * Переглядати каталог усіх завантажених ігор.
  * **Додавати** нові ігри через HTML-форму.
  * **Редагувати** існуючі записи (Назва, Жанр).
  * **Видаляти** ігри з бази даних.

# Software Documentation and Design Patterns 

Репозиторій містить комплексний проєкт, розроблений в рамках курсу "Документування програмного забезпечення та шаблони проєктування". Проєкт демонструє еволюцію від проєктування архітектури (UML) до реалізації багаторівневого веб-додатка та використання сучасних патернів програмування і брокерів повідомлень.

Основна предметна область (Лабораторні 1-3): **Ігрова платформа Steam** (реєстрація користувачів, каталог ігор, ігрові сесії).  
Предметна область (Лабораторна 4): **Обробка відкритих даних** (NYPD Motor Vehicle Collisions).

---

## 🛠 Технологічний стек
* **Мова:** Python 3.10+
* **Веб-фреймворк:** FastAPI, Jinja2
* **Бази даних:** SQLite (SQLAlchemy ORM), Redis, Google Firestore (NoSQL)
* **Брокер повідомлень:** Apache Kafka
* **Інфраструктура:** Docker, Docker Compose
* **UI/UX:** HTML5, Bootstrap 5
* **Генерація даних:** Faker

## 📂 Опис лабораторних робіт

### Лабораторна робота 1: UML-моделювання
Проєктування архітектури системи "Steam" за допомогою уніфікованої мови моделювання.
* **Use Case Diagram:** Реєстрація користувача та купівля гри.
* **Class Diagram:** Об'єкти для пошуку гри, початку сесії та відслідковування статусу.
* **Activity Diagram:** Демонстрація ігрової сесії та запису її стану.
* **Sequence Diagram:** Процес періодичного збереження стану ігрової сесії в базу даних.

### Лабораторна робота 2: Багаторівнева архітектура (Layered Architecture)
Реалізація суворої трирівневої архітектури з використанням шаблонів **Інверсія управління (IoC)** та **Впровадження залежностей (DI)**.
* **DAL (Data Access Layer):** Робота з базою даних SQLite через SQLAlchemy ORM.
* **BLL (Business Logic Layer):** Бізнес-логіка, яка керує створенням об'єктів та парсингом даних, повністю ізольована інтерфейсами.
* **Presentation Layer:** API-ендпоінти для автоматизованого імпорту 1000+ денормалізованих рядків з CSV-файлу в реляційні таблиці.

### Лабораторна робота 3: Патерн MVC (Model-View-Controller)
Створення повноцінного веб-інтерфейсу для візуалізації та управління даними.
* Реалізовано контролери, які обробляють HTTP-запити від користувача.
* Створено HTML-представлення (Views) за допомогою шаблонізатора Jinja2 та Bootstrap.
* Реалізовано повний цикл **CRUD (Створити, Читати, Оновити, Видалити)** для сутності "Гра" через веб-форми.

### Лабораторна робота 4: Патерн Стратегія (Strategy Pattern)
Відокремлення логіки вичитки даних від логіки їх збереження. Демонстрація роботи з різними сховищами "на льоту".
* Читання реальних даних (Нью-Йоркські ДТП). Посилання на датасет [тут](https://data.cityofnewyork.us/Public-Safety/Motor-Vehicle-Collisions-Vehicles/bm4k-52h4/about_data)
* Динамічне перемикання місця збереження результатів за допомогою файлу `config.json` без зміни основного коду.
* **Реалізовані стратегії:**
  * `ConsoleStrategy` (форматований вивід на екран).
  * `RedisStrategy` (запис у In-memory БД).
  * `KafkaStrategy` (публікація повідомлень у топік).
  * `FirestoreStrategy` (збереження у хмарну NoSQL базу Google).

## 🚀 Інструкція із запуску

### Загальна підготовка (Для всього проєкту)
1. Склонуйте репозиторій та перейдіть у папку проєкту.
2. Створіть та активуйте віртуальне середовище Python.
3. Встановіть необхідні залежності:
   ```bash
   pip install -r requirements.txt
   ```

### Запуск Лабораторних 2 та 3 (Ігрова платформа)
*Ці роботи виконуються в основній директорії проєкту.*

1. **Генерація даних:** Створіть тестовий CSV-файл (1000+ рядків):
   ```bash
   python data_generator/generate_csv.py
   ```
2. **Запуск веб-сервера:** База даних `steam_app.db` створиться автоматично.
   ```bash
   python main.py
   ```
3. **Імпорт даних (Лаба 2):** Відкрийте новий термінал та запустіть клієнтський скрипт для масового запису даних у БД:
   ```bash
   python client_test.py
   ```
4. **Веб-інтерфейс (Лаба 3):** Відкрийте браузер за адресою [http://127.0.0.1:8000/games](http://127.0.0.1:8000/games) для доступу до каталогу ігор та використання CRUD-операцій.

### Запуск Лабораторної 4 (Патерн Стратегія)
*Ця робота виконується у виділеній папці (наприклад, `strategy_pattern/`).*

1. **Налаштування середовища:** Створіть файл `.env` для підключення до Google Firestore (додайте свій `firebase_key.json`):
   ```env
   FIRESTORE_PROJECT_ID=ваші-дані
   FIRESTORE_COLLECTION=nyc_collisions
   FIRESTORE_KEY_PATH=firebase_key.json
   ```
2. **Запуск інфраструктури:** Підніміть контейнери Redis та Kafka за допомогою Docker:
   ```bash
   docker-compose up -d
   ```
   *(Зачекайте 15-20 секунд для повної ініціалізації Kafka).*
3. **Вибір стратегії:** Відкрийте файл `config.json` та вкажіть бажаний тип сховища в полі `storage_type` (варіанти: `"console"`, `"redis"`, `"kafka"`, `"firestore"`).
4. **Виконання:** Запустіть головний скрипт:
   ```bash
   python main.py
   ```
5. **Зупинка інфраструктури:** Після завершення роботи вимкніть Docker-контейнери:
   ```bash
   docker-compose down
   ```

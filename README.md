# Steam Platform Data Management System (Лабораторна 2)

Проєкт для автоматизації генерації, обробки та імпорту денормалізованих даних ігрової платформи (користувачі, ігри, ігрові сеанси, досягнення) з CSV-файлу в реляційну базу даних. Проєкт побудований з використанням суворої трирівневої архітектури, шаблонів **Інверсія управління (IoC)** та **Впровадження залежностей (DI)**.

## Технології

* **Мова:** Python 3.10+
* **Framework:** FastAPI (веб-сервер та DI-контейнер)
* **ORM:** SQLAlchemy 2.0+
* **База даних:** SQLite (вбудована, драйвер `sqlite3`)
* **Формат даних:** єдиний денормалізований `.csv` файл (1000+ рядків)
* **Генерація даних:** Faker
* **Сервер:** Uvicorn

## Архітектура проєкту

Проєкт реалізований за принципом **Layered Architecture** (Багатошарова архітектура). Бізнес-логіка повністю ізольована від деталей реалізації бази даних та веб-фреймворку за допомогою інтерфейсів (абстрактних класів).

### Структура шарів:

**1. DAL (Data Access Layer) — Рівень доступу до даних (`data_access/`)**
* `interfaces.py`: Містить абстрактні класи (`IDataLoader`, `IPlatformRepository`), які визначають контракти для роботи з даними.
* `repositories.py`: Інкапсулює логіку запитів до бази даних через SQLAlchemy ORM.
* `csv_reader.py`: Відповідає за читання сирих даних з файлової системи.
* `models.py`: ORM моделі таблиць бази даних із налаштованими зв'язками (Foreign Keys).

**2. BLL (Business Logic Layer) — Рівень бізнес-логіки (`business/`)**
* `services.py`: Головний сервіс `SteamPlatformService`. Координує процес: розв'язує денормалізовані рядки з CSV, створює унікальні об'єкти (User, Game), формує зв'язки для сеансів та досягнень, після чого передає їх у репозиторій для збереження. **Залежить виключно від інтерфейсів.**
* `dependencies.py`: Налаштування Dependency Injection за допомогою механізму `Depends()` з FastAPI.

**3. Presentation Layer — Презентаційний рівень (`presentation/` та `main.py`)**
* `routers.py`: FastAPI ендпоінти. Не містять бізнес-логіки, лише приймають запити та викликають відповідні сервіси.
* `main.py`: Точка входу, ініціалізація бази даних та налаштування сервера. (Увага: Swagger UI навмисно вимкнено для імітації production-середовища).

## Інструкція із запуску

### 1. Встановлення залежностей
Переконайтеся, що ви знаходитесь у віртуальному середовищі проєкту. Встановіть всі необхідні бібліотеки з файлу `requirements.txt`:
```bash
pip install -r requirements.txt
```

### 2. Генерація тестових даних
Для роботи програми необхідний CSV-файл з даними. Запустіть скрипт-генератор, який створить файл `platform_data.csv` (понад 1000 рядків) у папці `data/`:
```bash 
python data_generator/generate_csv.py
```

### 3. Запуск сервера
Запустіть основний додаток за допомогою Uvicorn. База даних `steam_app.db` (SQLite) створиться автоматично під час першого запуску:
```bash
python main.py
```

### 4. Тестування та імпорт даних
Для тригеру процесу імпорту використовується клієнтський скрипт. Не зупиняючи сервер, відкрийте новий термінал і виконайте:
```bash
python client_test.py
```
Цей скрипт відправить POST-запит на сервер, який ініціює читання CSV-файлу, парсинг даних через рівень бізнес-логіки та їх збереження у базу даних. Результати можна переглянути за допомогою будь-якого клієнта бази даних (наприклад, DBeaver), підключившись до файлу `steam_app.db`.
